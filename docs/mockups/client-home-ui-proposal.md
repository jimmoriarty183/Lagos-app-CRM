# UI proposal: главная страница клиента (Orders / Business / Analytics)

> Контекст: предложен редизайн домашнего экрана клиента с фокусом на быстрый обзор заказов, понятные действия и удобство на desktop + mobile.

## 1) UX-цели

1. Клиент за 3–5 секунд понимает: сколько заказов, что просрочено, что нужно оплатить.
2. Основное действие (добавить/найти/обновить заказ) доступно без скролла на desktop.
3. В мобильной версии пользователь работает «большим пальцем»: фильтры, карточки, быстрое изменение статуса.
4. Блок Business не отвлекает от заказов, но дает прозрачный доступ к ролям и управлению командой.

---

## 2) Информационная архитектура экрана

1. **Top bar**
   - выбор бизнеса
   - уведомления
   - профиль/выход
2. **KPI summary**
   - Total orders
   - Active amount
   - Overdue
   - Waiting payment
3. **Action row**
   - CTA: Add order
   - Search + Filters
4. **Orders list**
   - desktop: таблица
   - mobile: карточки
5. **Business panel**
   - роли
   - участники
   - manage access

### Desktop UX-решения

- Левую колонку оставить sticky (Business + команда), чтобы контекст бизнеса не терялся.
- KPI-карточки объединить по приоритету: **Overdue** и **Waiting payment** визуально контрастнее.
- Фильтры в одну строку + фиксированная кнопка Apply.
- Таблица: закрепить заголовок и первую колонку `Client` при горизонтальном скролле.
- Для статусов использовать единый цветовой словарь:
  - NEW — нейтральный
  - IN PROGRESS — синий
  - DONE — зеленый
  - OVERDUE label — красный

- Основные действия в карточке: `View`, `Edit`, `Status` (inline).

## 5) Рекомендации по визуалу

- Сетка:
  - Desktop: 12 columns, max-width 1440, content gutter 24.
  - Mobile: 4 columns, side padding 16.
- Скругления: 14–16px для карточек и контролов.
- Тени: мягкие, одноуровневые (чтобы сохранить «чистый CRM»-стиль).
- Типографика:
  - KPI value: 32/700
  - Section title: 20/600
  - Body: 15–16/400
- Интерактивность:
  - hover (desktop), pressed (mobile), focus ring для accessibility.

---

## 6) Что лучше со стороны клиента (приоритет)

1. **Сначала заказы и критичные сигналы** (Overdue, Waiting payment).
2. **Минимум кликов до изменения статуса/оплаты**.
3. **Понятный мобильный сценарий**: поиск → фильтр → открыть карточку → изменить статус.
4. **Прозрачность по бизнесу и ролям** без перегруза основной зоны.
